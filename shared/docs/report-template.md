# OpenClaw 助手汇报模板（可复用）

目的：把每次任务的输出固定成可扫读的结构，方便 wik 快速查看进度与风险。

## 汇报结构（建议每次都按这个顺序）

### 1) 结论 / 当前状态

- 这次任务是否完成（完成/部分完成/卡住）
- 关键结果是什么（1-3 条）

### 2) 我做了什么（可追溯）

- 做了哪些步骤（按时间或按模块）
- 关键决策是什么（含理由）

### 3) 产出与变更（可复用资产）

- 新增/修改了哪些文件（路径列表）
- 新增的规范/模板/脚本（如何复用、如何验证）

### 4) 验证与证据

- 我如何验证它真的可用（命令/测试/截图/日志）
- 关键输出摘要（不要粘贴巨量日志）

### 5) 风险 / 未决项

- 现在的风险点是什么
- 还缺什么信息或授权

### 6) 下一步（给 wik 的选择题）

- 给出 1-3 个选项，让 wik 用数字回复即可推进

---

## 可直接复制的最小模板

```text
【任务】

【结论/状态】
- 

【我做了什么】
- 

【产出与变更】
- 

【验证】
- 

【风险/未决】
- 

【下一步（选 1/2/3）】
1) 
2) 
3) 
```
