# OpenClaw 配置变更流程（可复用）

目标：避免把 OpenClaw 配置改挂；每次变更都可追溯、可回滚、可验收。

适用范围：

- `~/.openclaw/openclaw.json`（网关/渠道/工具策略/模型等）
- 任何会影响对外暴露、权限、鉴权、成本、稳定性的配置

---

## 1) 变更前（必做）

- 明确变更目标：要解决什么问题？成功标准是什么？
- 明确影响面：是否影响鉴权、端口暴露、消息渠道、工具权限、费用。
- 准备回滚：确保有备份，且知道如何恢复。

## 2) 变更步骤（建议顺序）

1. 备份配置（带时间戳）
2. 做最小改动（一次只改一类东西：鉴权/渠道/工具/模型/网络）
3. 做静态校验（JSON 可解析）
4. 跑安全审计（`openclaw security audit`）
5. 做最小验证（启动/探活/关键功能调用 1 次）

> 原则：不要在一次提交里混合多个不相关的大改动。

## 3) 验收清单

- `openclaw security audit` 无新增 critical
- 关键渠道可用（例如 Feishu 可正常收发）
- 关键工具可用（例如 MCP/读写文件/浏览器等）
- 成本/限额策略没有被意外放宽（如有）

## 4) 回滚策略

- 发现异常（启动失败/反复重启/功能不可用）时：
  - 立刻把 `~/.openclaw/openclaw.json` 回滚到最近一次备份
  - 重新启动/重新验证

## 5) 推荐：用脚本固化流程

仓库提供了 `scripts/config-guard.sh` 用于：

- 自动备份
- JSON 校验
- 安全审计
- 输出下一步提示

用法见脚本头部说明。
