# OpenClaw Agent 工作流规范（可复用）

这份文档定义一套可复用的「养 Agent」工作流：如何防失忆、怎么写上下文、怎么授权、如何迭代技能、怎么做复盘。

目标：减少重复解释、减少跑偏、提高自动化成功率，同时把高风险操作锁死。

## 1. 文件约定（单实例也能跑）

- `USER.md`：用户画像（你是谁 + 偏好 + 时区）
- `IDENTITY.md`：Agent 身份（名字/性格/语气/输出风格）
- `LEARNING.md`：可执行的“规矩”（错一次写一条，防止重复犯错）
- `SESSION.md`：复杂任务的“现场指挥板”（多轮/多小时任务必备）
- `memory/YYYY-MM-DD.md`：每日流水账（关键决策/进度/遗留）

> 原则：重要信息必须落到文件里。不要靠“我记住了”。

## 2. 防失忆：把“为什么”写下来

### 2.1 Daily log：`memory/YYYY-MM-DD.md`

写什么：

- 今天做了什么（只记关键点）
- 哪些决策已敲定（以及理由）
- 遗留/风险/明天继续的点

不要写：长篇流水、复制粘贴一堆输出。

### 2.2 长任务：`SESSION.md`

当任务满足任一条件，就必须建 `SESSION.md`：

- 会持续 30 分钟以上
- 涉及多文件/多模块
- 会改配置/改部署/改权限

`SESSION.md` 固定四块：

1) 当前目标
2) 已做关键决策（含理由）
3) 明确否掉的方案（含理由）
4) 当前进度与下一步

> 频率：每 20-30 分钟更新一次，2 分钟写完。

### 2.3 主动压缩：别等系统自动 compaction

当上下文变长（信息开始重复、模型开始“忘了为什么”），主动触发压缩，并明确告诉它必须保留：

- 关键决策 + 理由
- 否决方案 + 原因
- 任何安全边界（不要做什么）

## 3. 身份与用户画像：减少反复纠正

### 3.1 `IDENTITY.md` 要具体

- 风格标签：`sharp`/`warm`/`calm`/`chaotic`（选 1-2 个）
- 输出偏好：要不要 Markdown、要不要表格、要不要先结论

### 3.2 `USER.md` 必填字段

- 时区（提醒/日志必须正确）
- 写作/表达偏好（实用主义/口语化/结构化等）
- 禁忌项（不想看到的表达方式/格式）

## 4. 授权策略：低风险放开，高风险锁死

- 低风险（建议默认放开）：读文件、写 Markdown、运行只读诊断命令、安装“业务必需”的依赖
- 高风险（必须确认）：删除/覆盖重要文件、修改主配置、对外发送消息/发布、重启关键服务

> 规则：让 Agent 在安全边界内“放手干”，不要每一步都确认。

## 5. Skills 迭代：小步快跑 + 可验证

每新增一个能力（Skill/MCP/脚本），必须同时提供：

- 安装步骤（可重复执行）
- 验证步骤（一条命令或一个最小用例）
- 失败时的排障提示（常见报错 + 修复）

## 6. 训练闭环：写进 `LEARNING.md`

当出现以下情况，立刻写规则：

- 同类错误出现 2 次
- 输出风格反复跑偏
- 关键安全边界被触碰

规则格式建议：

- 触发条件：什么时候算“要执行这条规则”
- 行为约束：必须做/必须不做
- 验证方式：怎么确认做对了

## 7. 配置修改安全（防自毁）

- 改配置前：先读官方文档确认字段层级/示例
- 改配置时：尽量小改动、可回滚
- 改完后：立刻做一次最小验证（启动/健康检查/一条调用）
