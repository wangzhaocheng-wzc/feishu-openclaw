# SESSION.md (示例) - 学习/接入新技能

## 当前目标

- 学会并接入一个新能力（Skill/MCP/脚本/工具），做到“可复现安装 + 可验证可用 + 可复用文档”。

## 关键决策（含理由）

- 先做最小验证用例（MVP），再做工程化（文档/模板/自动化）。
  - 理由：先确认能用，避免投入后发现走不通。
- 安装步骤必须可重复执行；验证步骤必须是一条命令或一个最小用例。
  - 理由：方便排障与回归测试。

## 明确否决的方案（含理由）

- 不依赖不稳定的外部下载（例如 CDN 拉浏览器包）作为关键路径。
  - 理由：在服务器环境容易失败；优先系统包/镜像。
