# LEARNING.md

这里放“可执行的规矩”，用于避免重复犯错。

写法：一次错误 -> 追加一条规则（包含触发条件/行为约束/验证方式）。

更系统的工作流规范见：`docs/agent-workflow.md`。

---

## 规则列表

- 触发条件：复杂任务（>30min / 多文件 / 可能触发压缩）
  行为约束：必须创建并维护 `SESSION.md`（目标/决策+理由/否决项+理由/进度+下一步）。
  验证方式：中途发生 compaction 后，仍能快速恢复“为什么这么做”。

- 触发条件：上下文明显变长（重复解释、开始忘理由）
  行为约束：主动 compact，并明确保留“关键决策+理由”“否决方案+原因”“安全边界”。
  验证方式：compact 后不会把已否决方案又捡回来。

- 触发条件：需要提权/安装依赖
  行为约束：优先选择可复现、可验证的安装方式；同时给出最小验证用例。
  验证方式：一条最小命令即可证明能力可用。
